## 2. フィーチャー定義

### 2.1 F-004: 時系列分析（Temporal Analysis）

#### 2.1.1 概要

知識グラフ内のエンティティと関係に時間属性を付与し、AI研究トレンドの時系列変化を分析・可視化します。

#### 2.1.2 ユースケース

| UC-ID | ユースケース | アクター |
|-------|--------------|----------|
| UC-004-1 | 特定技術の採用トレンドを時系列で表示 | AI研究者 |
| UC-004-2 | 年代別の主要論文・モデルを一覧表示 | AI研究者 |
| UC-004-3 | 技術の「ブレイクスルー」時点を特定 | PM/ビジネスリーダー |
| UC-004-4 | 研究トレンドの予測（次期トレンド） | AI研究者 |
| UC-004-5 | 特定期間のHot Topics抽出 | 全ユーザー |

#### 2.1.3 要件（EARS形式）

**機能要件**

| REQ-ID | 種別 | 要件 |
|--------|------|------|
| REQ-004-01 | Ubiquitous | The system SHALL store temporal metadata (publication_date, first_cited_date) for all entities. |
| REQ-004-02 | Event-driven | WHEN a new paper is ingested, the system SHALL extract and index temporal information. |
| REQ-004-03 | Ubiquitous | The system SHALL provide time-range filtering for all graph queries. |
| REQ-004-04 | State-driven | WHILE analyzing trends, the system SHALL calculate adoption_rate, citation_velocity, and momentum metrics. |
| REQ-004-05 | Optional | WHERE historical data exceeds 3 years, the system MAY generate trend forecasts using time-series models. |
| REQ-004-06 | Ubiquitous | The system SHALL provide timeline visualization data in JSON format for frontend consumption. |
| REQ-004-07 | Event-driven | WHEN queried for "hot topics", the system SHALL return top-N entities by momentum score for the specified period. |

**非機能要件**

| REQ-ID | 種別 | 要件 |
|--------|------|------|
| REQ-004-N01 | Performance | Time-range queries SHALL complete within 500ms for 100,000 entities. |
| REQ-004-N02 | Accuracy | Trend detection SHALL achieve >85% precision compared to manual expert labeling. |

#### 2.1.4 CLIコマンド定義（Article II準拠）

```bash
# タイムライン表示
yagokoro temporal timeline <entityType> --start <year> --end <year> --limit <n>
yagokoro temporal timeline AIModel --start 2020 --end 2024 --limit 20

# トレンド検出
yagokoro temporal trends --period <year> --type <entityType> --top <n>
yagokoro temporal trends --period 2024 --type Technique --top 10

# Hot Topics
yagokoro temporal hot --period <year> --category <category>
yagokoro temporal hot --period 2024 --category "Large Language Models"

# トレンド比較
yagokoro temporal compare <entity1> <entity2> --metric <metric>
yagokoro temporal compare "GPT-4" "Claude" --metric citations
```

#### 2.1.5 技術設計方針

```
@yagokoro/temporal (新規パッケージ)
├── TemporalIndexer      # 時間属性のインデックス構築
├── TrendAnalyzer        # トレンド分析（momentum, velocity）
├── TimelineGenerator    # タイムライン可視化データ生成
├── HotTopicDetector     # Hot Topics検出
└── TrendForecaster      # トレンド予測（オプション）
```

### 2.2 F-005: 研究者コラボレーション分析（Researcher Network）

#### 2.2.1 概要

研究者（Person）エンティティ間の共著関係、所属組織の関係、引用ネットワークを分析し、研究コミュニティの構造を可視化します。

#### 2.2.2 ユースケース

| UC-ID | ユースケース | アクター |
|-------|--------------|----------|
| UC-005-1 | 特定研究者の共著ネットワークを表示 | AI研究者 |
| UC-005-2 | 研究コミュニティ（クラスタ）を検出 | AI研究者 |
| UC-005-3 | 組織間のコラボレーション関係を分析 | PM/ビジネスリーダー |
| UC-005-4 | 影響力の高い研究者をランキング | 全ユーザー |
| UC-005-5 | 2人の研究者間の関係パスを探索 | AI研究者 |
| UC-005-6 | 研究者のキャリア推移を表示 | AI研究者 |

#### 2.2.3 要件（EARS形式）

**機能要件**

| REQ-ID | 種別 | 要件 |
|--------|------|------|
| REQ-005-01 | Ubiquitous | The system SHALL extract and store co-author relationships from papers. |
| REQ-005-02 | Ubiquitous | The system SHALL detect and store researcher affiliation history over time. |
| REQ-005-03 | Event-driven | WHEN analyzing a researcher, the system SHALL calculate h-index, citation_count, and collaboration_score. |
| REQ-005-04 | Ubiquitous | The system SHALL detect researcher communities using Leiden/Louvain algorithm. |
| REQ-005-05 | State-driven | WHILE building the collaboration graph, the system SHALL weight edges by co-author frequency and recency. |
| REQ-005-06 | Ubiquitous | The system SHALL provide shortest-path analysis between any two researchers. |
| REQ-005-07 | Optional | WHERE ORCID data is available, the system MAY enrich researcher profiles with verified identities. |
| REQ-005-08 | Event-driven | WHEN queried for "top researchers" in a field, the system SHALL return ranked list by influence score. |

**非機能要件**

| REQ-ID | 種別 | 要件 |
|--------|------|------|
| REQ-005-N01 | Scalability | The system SHALL handle >50,000 researcher nodes without degradation. |
| REQ-005-N02 | Privacy | The system SHALL only use publicly available information (papers, affiliations). |

#### 2.2.4 CLIコマンド定義（Article II準拠）

```bash
# 研究者プロファイル
yagokoro researcher profile <name> [--orcid <orcid>]
yagokoro researcher profile "Ilya Sutskever"
yagokoro researcher profile --orcid "0000-0001-2345-6789"

# 共著ネットワーク
yagokoro researcher network <name> --depth <n> --min-papers <n>
yagokoro researcher network "Yann LeCun" --depth 2 --min-papers 3

# 研究者間パス探索
yagokoro researcher path <researcher1> <researcher2> --max-hops <n>
yagokoro researcher path "Geoffrey Hinton" "Andrej Karpathy" --max-hops 4

# 研究者ランキング
yagokoro researcher rank --field <field> --metric <metric> --top <n>
yagokoro researcher rank --field "LLM" --metric h-index --top 20

# コミュニティ検出
yagokoro researcher communities --field <field> --min-size <n>
yagokoro researcher communities --field "Vision Transformer" --min-size 5

# キャリアタイムライン
yagokoro researcher career <name>
yagokoro researcher career "Demis Hassabis"
```

// libs/temporal/src/__tests__/temporal.indexer.test.ts

describe('TemporalIndexer', () => {
  // REQ-004-01: The system SHALL store temporal metadata for all entities
  describe('REQ-004-01: Temporal Metadata Storage', () => {

    });

      expect(result.publicationDate).toBeDefined();

  });
});

describe('TrendAnalyzer', () => {
